Pie Eater

by: Ariana

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I wanted to make a simple game which had a moving player and it would collect items like in Snake. I picked pies as the target object as I waned to the general feel of the game to be cozier.

2. What I Learned

I learned from user testing that my game ran differently on different computers. I also learned that without instructions it was difficult to understand the rules. One major change was removing a level as my main tester found it repetitive.

3. Architecture of My Game

The Pie and RocketShip classes interact the most. Pie objects are made in an ArrayList and whenever the RocketShip's collision box and the Pie collision box collide, the Pie object is removed from the list.

4. Technical Implementation

I had so much trouble with a certain part of the game that I'm most proud of the game actually working. I had many external things that I had to take care that I nearly forgot to fix the score system in the game. One of the problems with the game also included breaking after winning the game. This was due to an if statement that looked if all the pies were eaten. This was meant to be used for the level system but it was removed due to my Tester saying it was too repetitive.

5. Most Challenging Part

One of the hardest parts of the game was making it so that the pie would disappear when touched. When debugging, I knew that the game wasn't entering an if statement that would control score and the pies disappearing. But I never find the problem in the if statement's logic. After a while, I realized the problem was in the RocketShip class, explaining how one part of the code worked but code including/dependent on the rocket would fail.

6. Future Fixes and Updates

I would like to make the game more complex, with an enemy system and better backgrounds. It would be interesting to have a health system for the rocket as well.
